# DEVELOPMENT_PLAN.md

## 项目名称

Clothing Color Match Studio

## 产品定位

一个用于服装样品图、电商图、模特图、面料图的自动校色在线网站。

核心目标：

- 通过上传标准图建立校色参考
- 对待校色服装图片进行自动校色
- 只改变衣服颜色
- 保留衣服纹理、褶皱、光影、图案和面料细节
- 支持蒙版编辑、人工精修、实时预览、前后对比、批量导出
- 支持本地 HTML 验证和后续部署到 GitHub / Vercel

---

## 核心功能清单

### 1. 标准图上传

- 上传一张标准图
- 用户选择标准图中的衣服区域
- 从标准图衣服区域提取参考颜色
- 支持重新上传和重新选择参考区域

### 2. 样品图上传

- 上传单张样品图
- 批量上传样品图
- 支持 JPG / PNG / WebP
- 显示样品图缩略图
- 点击缩略图切换当前编辑图片

### 3. 蒙版编辑

- 画笔添加蒙版
- 橡皮擦删除蒙版
- 画笔大小调整
- 蒙版透明度调整
- 蒙版显示 / 隐藏
- 清空蒙版
- 撤销
- 重做
- 放大 / 缩小
- 平移画布
- 蒙版边缘羽化

### 4. 自动校色

- 基于标准图衣服区域提取颜色
- 基于待处理图衣服蒙版区域进行校色
- 使用 Lab 色彩空间
- 保留目标图 L 通道
- 主要迁移 a / b 通道
- 支持校色强度
- 支持高光保护
- 支持阴影保护
- 支持纹理保留

### 5. 人工调整

- 即时修复曝光不足
- 增强色彩
- 校正白平衡
- 亮度
- 对比度
- 饱和度
- 色相
- 曝光
- 阴影
- 高光
- 色温
- 色调
- 清晰度
- 校色强度
- 纹理保留强度

### 6. 实时预览和前后对比

- 实时预览处理结果
- 原图 / 结果图左右对比
- 拖动分割线对比
- 按住快捷键查看原图
- 放大查看衣服纹理细节

### 7. 批量处理

- 批量应用当前标准图和参数
- 批量处理进度显示
- 单张失败提示
- 失败重试
- 单张下载
- 批量 ZIP 下载

### 8. 分辨率导出

- 原尺寸导出
- 2K 导出，长边 2048px
- 4K 导出，长边 4096px
- 保持原始比例
- 不允许拉伸变形
- 文件名自动添加后缀

### 9. 本地 HTML 和在线部署

- 支持本地运行和交互验证
- 支持团队分享验证
- 后续可上传 GitHub
- 支持部署到 Vercel / Netlify / GitHub Pages

# 开发阶段

## 阶段 1：基础项目和页面框架

### 目标

- 搭建 React + Vite + TypeScript 项目
- 引入 Tailwind CSS
- 完成基础 UI 布局
- 页面包含：
  - 顶部工具栏
  - 左侧图片列表
  - 中央 Canvas 预览区
  - 右侧参数调整区
  - 底部导出区

### 验收标准

- npm run build 成功
- 页面可以正常打开
- UI 结构清晰
- 不需要实现真实图像处理功能

---

## 阶段 2：图片上传和预览

- 支持上传标准图
- 支持批量上传样品图
- 显示缩略图列表

- Canvas 中显示当前图片
- 图片按比例显示

### 验收标准

- JPG / PNG / WebP 可上传
- 多图可以正常切换
- 图片显示不变形
- 上传失败有提示

## 阶段 3：蒙版编辑

### 目标

- 支持蒙版透明度调整

- 支持清空蒙版

## 阶段 4：Lab 自动校色算法

### 目标

- 标准图和目标图都只分析蒙版区域
- 保留目标图 L 通道
- 迁移 a / b 色彩通道
- 支持校色强度
- 支持高光保护
- 支持阴影保护
- 支持蒙版边缘羽化

### 验收标准

- 校色只影响蒙版区域
- 背景和皮肤不变
- 衣服纹理、褶皱、光影没有明显丢失
- 不使用简单 RGB 替换
- 不使用整图滤镜

# 推荐开发节奏

```txt
读取计划
↓
执行单一 Task
↓
运行 npm run build
↓
人工验收
↓
修复问题
↓
Git commit
↓
进入下一个 Task
```

不要一次性让 Codex 完成整个项目。  
每个阶段稳定后再进入下一阶段。
