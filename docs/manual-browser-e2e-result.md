# Manual Browser E2E Result

## 1. 验收背景

本轮记录真实浏览器人工 E2E 验收结果，用于补充 `docs/e2e-release-acceptance-checklist.md` 中尚未覆盖的实际浏览器操作与视觉检查。

本阶段只记录结果，不修改前端、后端、算法、阈值、模型或导出逻辑。

## 2. 当前稳定版本

- Stable tag: `stable-frontend-color-transfer-safety-20260616`
- Branch: `docs/manual-browser-e2e-result`
- 相关安全基线：
  - target remote AI blocked / failed 不进入不安全 fallback
  - ROI / mask 变化后清理旧 `processedImages` / `adjustedImages`
  - blocked / failed mask 不应进入 `colorTransfer`
  - batch export 不应复用已失效结果

## 3. 验收方式

人工验收步骤覆盖：

1. 启动前端。
2. 在浏览器中打开项目页面。
3. 上传样本图进行真实浏览器测试。
4. 尝试远程 AI 自动识别服装区域。
5. 对高风险挂拍图使用手动蒙版选中服装区域。
6. 执行校色并观察校色前后结果。
7. 检查是否出现明显色块、边缘污染或旧结果复用迹象。

## 4. 真实浏览器测试结果

| 验收项 | 结果 |
|---|---|
| 前端启动 | 通过 |
| 浏览器打开项目 | 通过 |
| 上传样本图 | 通过 |
| 远程 AI 自动识别 | 可调用，但高风险挂拍图识别不够精准 |
| 手动蒙版路径 | 可用 |
| 手动蒙版校色后视觉结果 | 未观察到明显色块 |
| blocked / stale result 安全链路 | 基本有效 |

## 5. 远程 AI 自动识别表现

本轮人工验收确认：

- 自动 AI mask 在挂拍裤子、衣架、金属夹具、边缘贴图这类场景仍不够精准。
- 高风险挂拍图中，远程 AI 对裤子主体、衣架、金属杆、夹具边缘的区分仍不稳定。
- 这些场景不适合直接宣传为完全自动识别。
- 当前 remote AI 更适合作为辅助识别能力，而不是无人干预的最终服装分割能力。

结论：

> 当前 AI 自动识别精度不能作为完全自动化上线能力。它可以降低部分手动工作量，但高风险图仍需要手动蒙版兜底。

## 6. 手动蒙版校色表现

人工验收中，高风险挂拍图改用手动蒙版选中服装区域后：

- 校色流程可以正常执行。
- 校色区域由手动蒙版控制。
- 校色前后没有观察到明显矩形色块、局部青绿色色块或背景大面积污染。
- 说明当前 `colorTransfer` 在有效 mask 下的安全链路基本可用。

## 7. 是否出现色块问题

本轮人工验收结论：

- 使用远程 AI 自动识别时，高风险图的 mask 精度仍可能不足，因此不应直接进入生产式自动校色。
- 使用手动蒙版后，本轮没有观察到明显色块问题。
- 没有发现 ROI / mask 修改后继续复用旧 processed / adjusted result 的明显迹象。

## 8. 是否存在 blocked / stale export 风险

结合本轮人工验收和现有安全修复文档：

- target blocked / failed 后不应继续进入不安全 fallback。
- ROI / mask 变化后旧 `processedImages` / `adjustedImages` 会清理。
- 手动蒙版路径下校色结果与当前 mask 一致。

本轮未观察到 blocked mask 继续进入校色或 stale result 被导出的现象。

仍需注意：

- 本轮不是大规模样本池测试。
- 对于 hanger / metal / clip / edge-touching 等高风险图，仍建议人工复查 mask 后再导出。

## 9. 当前产品可用结论

当前版本可以作为：

> AI 自动识别辅助 + 手动蒙版兜底的 MVP / 内部验收版本。

当前版本不建议宣传为：

> 完全自动识别、无需人工干预的服装校色系统。

适合当前阶段的产品定位：

- AI 尝试生成初始 mask。
- 如果 AI mask 低置信、不完整、污染背景或命中道具，则阻断或提示。
- 用户通过手动蒙版修正服装主体。
- 在有效 mask 下执行 Lab 校色和导出。

## 10. 当前限制

- 自动 AI mask 在挂拍裤子、衣架、金属夹具、边缘贴图场景仍不够精准。
- 高风险挂拍图仍需要手动蒙版选中服装区域后再进行校色。
- AI 自动识别不应被视为完全自动化上线能力。
- 复杂背景、道具遮挡、贴边主体、局部近景图仍需要人工判断。
- 本轮人工验收样本有限，不等同于 50-100 张真实业务样本回归。

## 11. 上线建议

建议作为内部验收 / MVP 版本继续推进，前提是产品说明中明确：

- AI 识别是辅助能力。
- 高风险图片需要手动蒙版兜底。
- 不可靠 AI mask 会被阻断或需要人工修正。
- 当前版本不承诺完全自动识别。

上线前建议继续保留以下安全策略：

- blocked / failed mask 不进入校色。
- ROI / mask 变化后清理旧结果。
- 用户可以随时切换到手动蒙版路径。
- 导出前对高风险图进行人工视觉检查。

## 12. 后续优化建议

### P0

- 保持 blocked / failed 不进入校色。
- 保持手动蒙版兜底可用。
- 不为了提高 success 率而放宽安全门。

### P1

- 对高风险图提示用户优先使用手动蒙版。
- 在 AI mask 低置信度或高风险类型时给出明显提示。
- 增加“AI 识别不准时请手动框选服装区域”的 UI 文案。

### P2

- 后续替换更强分割模型。
- 扩大 50-100 张真实样本回归。
- 针对 hanger / metal / clip 类目单独优化。

## 13. 结论

- 自动 AI mask 在挂拍裤子、衣架、金属夹具、边缘贴图这类场景仍不够精准。
- 手动蒙版路径可用，并且本轮没有观察到明显色块问题。
- 当前 blocked / stale result / colorTransfer 安全链路基本有效。
- 当前版本可以作为“AI 辅助 + 手动蒙版兜底”的 MVP / 内部验收版本。
- 不建议宣传为完全自动识别。
- 下一阶段如要提升自动化，需要更强分割模型或更大样本回归，而不是继续小幅调安全门。
