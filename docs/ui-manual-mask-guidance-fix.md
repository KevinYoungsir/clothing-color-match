# UI Manual Mask Guidance Fix

## 1. 问题背景

`docs/manual-browser-e2e-result.md` 记录的真实浏览器 E2E 结果显示：

- 远程 AI 自动识别在挂拍裤子、衣架、金属夹具、边缘贴图等高风险场景仍不够精准。
- 使用手动蒙版选中服装区域后，校色前后没有观察到明显色块问题。
- 当前版本更适合定位为“AI 自动识别辅助 + 手动蒙版兜底”的 MVP / 内部验收版本，而不是完全自动识别产品。

本修复只增加前端提示文案，不修改算法、后端、质量门、`colorTransfer` 或导出逻辑。

## 2. 真实浏览器 E2E 结论

人工验收确认：

- AI mask 可以作为初始识别辅助。
- 挂拍、衣架、金属夹具和边缘贴图场景仍需要人工检查。
- 手动蒙版路径可用，并且本轮没有观察到明显色块问题。
- 不可靠 AI mask 应继续被 blocked / failed 阻断，不应进入校色。

## 3. 为什么需要增加手动蒙版提示

如果用户把远程 AI 识别理解为完全自动识别，就容易在高风险挂拍图中期待系统直接生成最终校色范围。当前实际能力更适合：

1. AI 生成初始蒙版。
2. 系统阻断明显不可靠的 mask。
3. 用户在复杂图上用手动蒙版修正最终校色范围。

因此 UI 需要更明确地告诉用户：AI 是辅助能力，手动蒙版是复杂场景的可靠兜底。

## 4. 修改了哪些 UI 文案

- 自动校色说明：明确 AI 识别仅作为辅助，并列出挂拍、衣架、金属夹具、边缘贴图等高风险场景。
- 远程 AI 识别说明：从“失败时回退传统识别”改为“低质量结果会被阻断，请用手动蒙版兜底”。
- 校色范围选项：将“智能识别服装”和“手动蒙版”的说明改为 AI 初始蒙版与手动最终兜底的关系。
- 自动识别失败 / blocked 后的 notice：提示用户使用手动蒙版修正后再校色。
- 蒙版面板：增加说明，指导用户用画笔添加服装区域、用橡皮擦去掉衣架/夹具/背景。
